Enterprise

Hylé for teams and organizations

Run Hylé at scale — private registries, self-hosted deployments, and org-scoped namespaces for your entire engineering org.

Three ways to deploy

Private Registry

Host your own print registry behind your firewall. Keep proprietary substrates private while still using the Hylé toolchain.

# .hyle/hyle.yaml
registry:
  url: https://registry.acme.corp
  auth: env:HYLE_REGISTRY_TOKEN
  • SSO / SAML authentication
  • Role-based access control
  • Audit logging to your SIEM
  • Custom domain + TLS

Self-hostable

Deploy the full Hylé registry stack as a single binary or Docker container. Air-gapped deployments supported.

# Binary
hyle-registry serve --port 8080

# Docker
docker run -p 8080:8080 \
  hyleorg/registry:latest
  • Single binary, no runtime deps
  • SQLite or Postgres storage
  • Air-gapped / offline support
  • Docker Compose examples

Org Namespace

Claim your org namespace on the public registry. All prints published under @acme/ are verified as belonging to your org.

# Publish under org namespace
hyle substrate publish \
  @acme/my-print

# Pull from org
hyle pull @acme/my-print
  • Verified org badge on all prints
  • Member management via GitHub org
  • Org-wide pull analytics
  • Custom branding support
